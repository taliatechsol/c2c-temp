Project Vision

Local-first, autonomous job-seeking engine

A zero-dependency desktop app that runs the end-to-end pipeline — Scout → Match → Tailor → Apply → Track — entirely on the user's machine. Tauri shell wraps a React UI and spawns a Python sidecar that hosts a LangGraph state machine, three embedded databases, an LLM router, and a Playwright browser actuator with a vision-language fallback.

Shell · Tauri 2.0

Rust process. Spawns Python sidecar, discovers its dynamic port, forwards events, owns the window.

tauri rust wix

UI · React + Vite

Five views (Dashboard, Pipeline, Knowledge, Activity, Profile) + Settings modal. Tailwind + Framer Motion.

react 19 framer tailwind 4

Brain · Python + LangGraph

FastAPI on a free port. APScheduler ghost-tick. Five agents wired into multiple LangGraph DAGs.

fastapi langgraph uv

Brawn · Playwright + VLM

Chromium drives DOM-fill on Lever/Greenhouse. Falls back to Claude vision (Set-of-Mark) on unknown forms.

playwright apify claude vision

§ 2 · Runtime view

End-to-end data flow

A user clicks Scan. Here's exactly what travels where, and which boxes touch which.

Cyan path · live feed

Every agent calls cm.broadcast({"type":"agent","event":"…","msg":"…"}). The UI's useWS() hook drops them into the Activity log and updates the spinner state.

Orange path · command + control

Buttons in the UI call REST endpoints. POST /api/v1/scan kicks the scout-then-evaluator background task. POST /api/v1/fire/{job_id} launches the actuator for a single approved lead.

§ 4 · The five hands

Agent modules

Each agent is a standalone Python file in backend/agents/. They share storage and the LLM router but never call each other directly — the LangGraph DAGs and the FastAPI background tasks compose them.

Ingestor · agents/ingestor.py   resumes → graph + vectors
Input
raw: str + optional PDF path. Reads PDF via pypdf.
LLM
call_llm(system, txt, C) — extracts skills, experience, projects into Pydantic C schema. Falls back to a hand-rolled key-value parser if no API key.
Graph writes
Creates fresh kuzu.Connection per call (avoids lock contention). Inserts Candidate, Skill, Experience, Project + WORKED_AS / BUILT / EXP_UTILIZES / PROJ_UTILIZES.
Vectors
SentenceTransformer all-MiniLM-L6-v2 (lazy-load with 15s timeout). Tables skills and projects.
Output
Returns the C profile object back to the caller (UI shows skill/role count).
Scout · agents/scout.py   queries/URLs → leads
Path A · Apify
If apify_token + apify_actor set, hits run-sync-get-dataset-items with {"queries":…}. Best for LinkedIn / X.
Path B · Local
Async Playwright Chromium → fetches HTML → html2text → LLM extracts {title, company, url, platform} via _Leads Pydantic model.
Dedup
md5 of URL = job_id; url_exists() short-circuits before LLM call.
Persistence
save_lead() → SQLite with default status='discovered'.
Evaluator · agents/evaluator.py   GraphRAG scoring
Proof of work
For each required skill, runs Cypher: MATCH (p:Project)-[:PROJ_UTILIZES]→(s:Skill {n:$n}) + experience variant. Concatenates titles, stacks, impacts.
Vector fallback
If Cypher returns nothing, embeds the skill name and runs k=3 LanceDB search on the skills table, then re-runs Cypher with the matched skill names.
Scoring
LLM returns {score: 0-100, reason, match_points[]}. ≥85 → tailoring, otherwise discarded. Status updated; events row written.
Generator · agents/generator.py   tailored PDF resume + cover
Inputs
Lead + match_points from evaluator + Cypher proof of work for the lead's skills.
Drafting
call_raw() (unstructured) — single Markdown blob with ## Resume and ## Cover Letter.
Render
markdown → HTML → fpdf.FPDF + HTMLMixin. Output: %LOCALAPPDATA%/BoomBoom/assets/<job_id>.pdf.
Status
save_asset_path() sets lead status approved.
Actuator · agents/actuator.py   the hands
Browser
Playwright Chromium · headless=False, slow_mo=80, real user-agent. Viewport 1280×900.
Primary fill
16-entry _DOM_MAP of CSS selectors → identity keys (first_name, email, phone, linkedin_url, …) + input[type='file'] for the resume PDF. _FILL_DELAY=500ms jitter.
Vision fallback
On any DOM exception → screenshot → Anthropic messages.parse with image + _Acts Pydantic schema. Returns ordered {kind: click|type, x, y, text}; replayed via p.mouse + p.keyboard.
Submit
Tries 6 selectors (button[type=submit], Submit Application, Apply, …). Honours dry_run by outlining the button red instead of clicking.
After
mark_applied() → status applied; events row written.
Shared · LLM router · backend/llm.py
call_llm
Returns Pydantic-typed object. Anthropic uses native messages.parse(output_format=…); Groq + NVIDIA + Ollama use instructor patches.
call_raw
Returns plain string for the generator's free-form Markdown.
Provider switch
Reads llm_provider from settings (sqlite). Missing key → falls back to local empty model construct, never throws.

Ghost mode
APScheduler · 6h tick

Ghost Mode

Settings → ghost_mode=true registers _ghost_tick on a 6-hour APScheduler interval. Each tick reads job_boards from settings and calls scout.run() with no human in the loop. Evaluator + generator + actuator follow if the score crosses the 85 threshold.

Sniper Mode

pipeline_graph is compiled with interrupt_before=["actuator"] + MemorySaver checkpointer. The graph runs through the generator, then halts; the user clicks Fire in the UI which calls POST /api/v1/fire/{job_id} to resume.

§ 9 · Choice of brain

LLM provider router

anthropic

claude-sonnet-4-6. Native messages.parse(output_format=…). Used by the Actuator's vision fallback (multimodal).

groq

llama-3.3-70b-versatile via OpenAI-compat. Wrapped with instructor for Pydantic responses.

nvidia

NIM endpoint, default z-ai/glm-5.1, JSON mode, 16k tokens, thinking disabled.

ollama (default)

Localhost :11434, model llama3. Zero-cost, fully offline path. Used when no key is set.

All callers pass a Pydantic BaseModel as the response shape. Missing-key paths return an empty model construct so a single agent never hard-crashes the pipeline.

§ 11 · Ergonomics

Where to make changes

Add a new agent

1) Drop agents/foo.py exposing a run(). 2) Wrap it in a LangGraph node in graph/nodes.py. 3) Expose a POST /api/v1/foo endpoint in main.py (BackgroundTask + cm.broadcast for live feed). 4) Add a button in App.tsx that calls fetch + listens for the event on the WS.

Change the graph schema

Edit the DDL list in db/client.py · _init(). Existing Kùzu DBs are forward-compatible thanks to IF NOT EXISTS — but new columns on existing node tables need a migration (delete %LOCALAPPDATA%/BoomBoom/graph or write a Cypher migration script).

Swap the LLM

Settings → LLM provider. Adding a new provider = ~25 lines in llm.py covering call_llm + call_raw. The agents don't care which one you use.

Tighten / loosen the score gate

The ≥ 85 threshold is hard-coded in two places: db/client.update_lead_score and graph/nodes.evaluator. Pull it into a setting (score_threshold) and read via get_setting if you want it tunable from the UI.

Add an application platform

Extend actuator._DOM_MAP with new selectors (most ATS use name="first_name" etc., already covered). For exotic forms, the vision fallback handles them automatically — but logging the screenshot to disk for review is a one-line add in _fill_vision.

Live-reload while developing

uv run python backend/main.py in one terminal, npm run tauri dev in another (Tauri auto-picks up the running backend's port). The UI hot-reloads on every App.tsx save.
